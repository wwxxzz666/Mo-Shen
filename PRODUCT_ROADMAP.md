# StoryAgents 产品开发路线图

这份路线图站在“用户是否愿意持续使用”的角度来排优先级，目标不是先堆功能，而是先补齐信任、可控性和长期创作体验。

## 核心目标

1. 先把产品从“能生成”提升到“敢长期写”。
2. 再把产品从“单次出稿工具”提升到“项目化创作工作台”。
3. 最后再做差异化能力，让用户愿意沉淀作品和流程。

## 排期总览

### Phase 1：本周可做

目标：先修用户最容易失望的地方，补齐“保存闭环”和“控制感”。

- 保存闭环
  - 编辑后的文本要能真正写回当前故事数据，而不只是替换页面展示。
  - 续写后的新章节要能回写原项目，而不是只流式展示。
  - 历史列表、查看详情、导出内容都应读取同一份最新结果。
- 模式切换
  - 增加 `快速出稿 / 标准创作 / 深度打磨` 三档。
  - 快速模式保留当前 4 节点流程。
  - 标准模式开启世界观、角色设计。
  - 深度模式再开启连续性审校和更多修订轮次。
- 章节级控制雏形
  - 先支持“只重写当前章”和“锁定已满意章节”。
  - 支持“先看大纲，再确认生成正文”。
- 历史记录基础增强
  - 支持重命名。
  - 支持复制为新项目。
  - 支持按标题和 prompt 搜索。

验收标准：

- 用户编辑一段文字后，刷新页面或从历史重新打开，仍能看到编辑后的结果。
- 用户续写后，历史记录中的章节数和正文同步增长。
- 不同模式下，界面上展示的产物与实际工作流保持一致，不出现空标签误导。

### Phase 2：两周内能做

目标：把“创作过程”变成真正可操控、可回看的工作台。

- 一致性面板
  - 常驻展示角色卡、世界规则、时间线、连续性备注。
  - 展示 showrunner handoff，告诉用户下一章是按什么思路接着写。
- 真正的编辑工作流
  - 展示原文和修改后版本对比。
  - 支持接受修改、撤销修改、另存为版本。
  - 支持对章节整体执行“改写 / 扩写 / 缩写 / 润色”。
- 章节工作区
  - 左侧章节目录。
  - 支持单章查看、单章导出、单章重写。
  - 支持锁章后只改后续章节。
- 更友好的模型设置
  - 默认提供 `省钱 / 平衡 / 质量优先` 三档。
  - 高级设置中再展开 provider、模型、输出语言等配置。
- 历史升级为项目库
  - 标签、置顶、归档。
  - 展示创建时间、章节数、模式、最后修改时间。

验收标准：

- 用户能清楚知道当前故事有哪些设定被继承。
- 用户可以只处理某一章，而不必每次整篇重来。
- 新用户不需要理解模型细节，也能根据目标做选择。

### Phase 3：后续规划

目标：做出区别于普通文本生成器的产品壁垒。

- 导入已有稿件
  - 导入用户已有章节，自动补齐大纲、人物、世界观。
  - 支持从已有稿件继续写，而不是必须从零开始。
- 多版本分支
  - 同一故事可以分叉出 A 线、B 线。
  - 支持对比两个剧情走向。
- 作者模板库
  - 用户保存自己的题材模板、节奏模板、角色模板。
  - 支持一键复用到新项目。
- 阅读与交付视图
  - 章节目录、字数统计、阅读模式、纸张排版。
  - 更完整的导出格式，比如按章节分页的 DOCX。
- 创作分析面板
  - 展示章节字数、重写次数、耗时、模型成本。
  - 帮助重度用户优化自己的创作流程。

## 推荐实施顺序

### 第一批

- 编辑结果持久化
- 续写结果回写项目
- 历史记录读取统一数据源

原因：这是信任问题，优先级最高。

### 第二批

- 模式切换
- 界面按模式动态展示产物
- 快速模式与标准模式的能力说明

原因：这是预期管理问题，能明显降低“为什么这个标签是空的”的困惑。

### 第三批

- 章节锁定
- 单章重写
- 大纲确认后再写正文

原因：这是控制感问题，决定用户会不会把它当成长期工具。

### 第四批

- 一致性面板
- 编辑对比视图
- 项目库标签与搜索

原因：这是效率问题，会直接提升长篇创作体验。

## 对应到当前项目的落地点

### 后端

- `storyagents/server.py`
  - 增加项目更新接口，而不只是新建和删除。
  - 为编辑、续写补齐真正的持久化写回逻辑。
- `storyagents/graph/setup.py`
  - 把 `fast_mode` 扩展为更清晰的模式枚举或档位配置。
- `storyagents/graph/story_graph.py`
  - 在流式事件里补齐更多可展示字段，比如世界观、角色卡、连续性备注。

### 前端

- `storyagents/h5/app.js`
  - 把当前前端局部替换式编辑，改成“修改后提交保存”流程。
  - 把续写、历史、导出统一到项目状态管理上。
  - 增加章节列表、模式开关、一致性面板。
- `storyagents/h5/index.html`
  - 为模式切换、章节目录、项目信息栏预留结构。

## 建议的迭代节奏

### 第 1 周

- 做保存闭环。
- 做模式切换。
- 隐藏无效空标签。

### 第 2 周

- 做章节目录和单章操作。
- 做一致性面板。
- 做编辑结果对比与保存。

### 第 3 周以后

- 做导入已有稿件。
- 做模板和分支。
- 做阅读视图和分析面板。

## 成功判断标准

- 用户第二次打开项目时，不会怀疑“我上次改的内容是不是丢了”。
- 用户能明确理解每个模式的差别，并能主动选择。
- 用户可以只改某一章，而不是被迫整篇重跑。
- 用户在长篇续写时，能看见并信任系统记住了哪些设定。
